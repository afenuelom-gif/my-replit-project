# AI Job Interview Platform

## Overview

A full-stack AI-powered job interview simulator. Users enter a job role and optional job description, then conduct a realistic 30-45 minute simulated interview with 2-3 AI interviewer avatars in a Zoom/Teams-style video call layout. At the end, a detailed performance report is generated.

## Architecture

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

### Stack
- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **AI**: OpenAI (GPT-4o for questions/evaluation/reports/posture, TTS for audio, gpt-4o-mini-transcribe for STT)
- **Frontend**: React + Vite + TailwindCSS + shadcn/ui
- **Auth**: Clerk (`@clerk/express` server-side, `@clerk/react` client-side)
- **Dev bypass**: Set `BYPASS_AUTH=true` in env to skip auth; attaches synthetic `dev_bypass_user`

## API Endpoints

- `GET /api/interview/interviewers` ‚Äî List all interviewer personas
- `POST /api/interview/sessions` ‚Äî Create a new interview session
- `GET /api/interview/sessions/:id` ‚Äî Get session with questions & interviewers
- `POST /api/interview/sessions/:id/next-question` ‚Äî Submit answer + get next question
- `POST /api/interview/sessions/:id/complete` ‚Äî End session, generate report
- `GET /api/interview/sessions/:id/report` ‚Äî Get performance report
- `POST /api/interview/tts` ‚Äî Text-to-Speech (OpenAI tts-1)
- `POST /api/interview/sessions/:id/transcribe` ‚Äî Speech-to-Text (gpt-4o-mini-transcribe)
- `POST /api/interview/sessions/:id/posture` ‚Äî Posture analysis via vision
- `GET /api/users/me` ‚Äî Get current user info (requires auth)
- `GET /api/users/me/sessions` ‚Äî Get user's interview history (requires auth)
- `GET /api/dev/status` ‚Äî Dev mode status (BYPASS_AUTH flag)

## Database Tables

- `users` ‚Äî Clerk user IDs, emails, session credits (for Stripe integration later)
- `interviewers` ‚Äî 6 interviewer personas with avatars, voices, personalities
- `interview_sessions` ‚Äî Session config (job role, desc, duration, status, userId FK)
- `interview_questions` ‚Äî All questions/answers per session
- `posture_analysis` ‚Äî Periodic webcam snapshots analyzed by vision AI
- `interview_reports` ‚Äî Generated performance reports (userId FK for history queries)

## Interview Flow

1. User enters job role + optional job description on home page
2. Session created ‚Üí 2-3 interviewers selected randomly
3. First question generated by GPT-4o based on job role
4. Questions auto-play via TTS with interviewer's voice
5. User answers by clicking mic button (MediaRecorder ‚Üí base64 ‚Üí STT transcription)
6. Answer sent to backend ‚Üí GPT-4o evaluates ‚Üí generates next question or follow-up
7. Posture captured via canvas snapshot every 90 seconds ‚Üí vision AI analysis
8. Session ends (timer or End Session button) ‚Üí GPT-4o generates full report
9. Report page shows overall score, category breakdown, per-question feedback, improvement tips
